KNN Classification

Predicts categorical class labels using majority voting among K nearest neighbors.

  • Output: Discrete class (e.g., Spam/Not Spam, A/B/C)
  • Rule: Predict the most frequent class among K neighbors
  • Weighted KNN: Closer neighbors get higher weight
  • Metrics: Accuracy, Precision, Recall, F1, Confusion Matrix

Classification Example

Email Classification (Spam vs Not Spam)

Training Email Feature 1: Capital % Feature 2: Links Label
Email 1 45% 2 Not Spam
Email 2 85% 8 Spam
Email 3 90% 7 Spam

New Email: 88% capitals, 6 links → K=3 finds 3 closest emails → Majority = Spam ✓

KNN Regression

Predicts continuous numeric values using average of K nearest neighbors' target values.

  • Output: Continuous number (e.g., price = $350,000)
  • Rule: Average or weighted average of K neighbors' target values
  • Formula: Prediction = (sum of K neighbors' values) / K
  • Metrics: MSE, MAE, R² (coefficient of determination)

Regression Example

House Price Prediction

Training House Area (sq ft) Bedrooms Price ($)
House 1 2000 3 300,000
House 2 2100 3 320,000
House 3 1950 3 310,000

New House: 2050 sq ft, 3 bedrooms → K=3 finds 3 closest → Price = (300K + 320K + 310K) / 3 = $310,000 ✓

Classification vs Regression

Classification

  • Output: Discrete class
  • Rule: Majority vote
  • Metrics: Accuracy, F1
  • Use: Labels, categories

Regression

  • Output: Continuous value
  • Rule: Average/mean
  • Metrics: MSE, R²
  • Use: Predictions, prices

Choosing K: Bias-Variance Trade-off

Small K (1, 3, 5)

High variance, low bias. Very flexible. Sensitive to noise. Risk of overfitting.

Large K (15, 25, 50)

Low variance, high bias. Smoother. Less sensitive to noise. May miss patterns.

Best Practice: Use cross-validation to try multiple K values and pick the best performing one.

Key Takeaways

  • KNN is simple, lazy, instance-based algorithm for classification & regression
  • Classification: predicts class labels via majority voting
  • Regression: predicts numeric values via averaging
  • Core concept: similar points are close in feature space
  • Feature scaling is critical to avoid bias
  • K selection involves bias-variance trade-off
  • Powerful for small/medium datasets; slow for large datasets
